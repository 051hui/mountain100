# 🏔️ 100대 명산 데이터 기반 개인화 등산로 추천 시스템
 분산된 등산 데이터를 통합·분석하여 사용자 맞춤형 등산로를 추천하는 데이터 기반 의사결정 플랫폼

---

## 📌 프로젝트 개요
공공데이터와 외부 API를 활용하여 흩어져 있는 등산 정보를 통합하고,  
사용자의 조건에 맞는 최적의 등산로를 추천하는 시스템을 구축한 프로젝트입니다.

👉 단순 정보 제공이 아닌 **데이터 기반 의사결정 지원 서비스 구현**을 목표로 진행되었습니다.

---

## ❗ 문제 정의
- 등산로 정보가 여러 플랫폼에 분산되어 있음
- 선택 기준이 명확하지 않아 개인 경험에 의존
- 탐색 과정이 복잡하여 계획 포기 발생

👉 **“사용자 조건에 맞는 등산로를 데이터로 추천할 수 있을까?”**

---

## 🎯 프로젝트 목표
- 등산로 데이터 통합 및 표준화
- 정량 지표 기반 비교 가능 구조 설계
- 개인 맞춤형 등산로 추천 시스템 구현
- 대시보드를 통한 직관적 의사결정 지원

---

## 📊 데이터

### 주요 데이터 소스
- 산림청 100대 명산 데이터
- GPX 등산로 좌표 데이터
- 전국 버스정류장 위치정보
- 관광명소 데이터
- 카카오맵 / 네이버 API (지도, 리뷰)
- 유튜브 / 블로그 후기 데이터

---

## ⚙️ 분석 과정

### 1. 데이터셋 구축
- GPX 데이터를 활용한 거리 계산 (Haversine)
- 고도 데이터 기반 누적 상승고도 계산
- Naismith 공식을 활용한 소요시간 추정
- 난이도 점수 및 등급 시스템 설계

👉 기존 데이터에 없는 핵심 정보 직접 생성

---

### 2. 데이터 전처리
- GPS 오류 제거 (경사도 기반 이상치 필터링)
- 결측치 처리 및 데이터 정합성 확보
- 좌표 기반 데이터 매칭 및 정제

---

### 3. 지표 설계

#### ✔️ 난이도 지수
- 거리 + 고도 + 경사도 기반 점수 산출
- 분위수 기반 등급화

#### ✔️ 관광 인프라 지수
- 반경 내 시설 수 + 거리 가중치 적용

#### ✔️ 접근성 지수
- 버스정류장 / 주차장 거리 기반 점수화

#### ✔️ 매력 지수 (NLP)
- 블로그, 리뷰, 댓글 데이터 수집
- LLM 기반 텍스트 필터링 및 감성 분석
- 키워드 추출 및 점수화

---

### 4. 군집 분석
- 5개 핵심 Feature 기반 클러스터링
- 등산로 유형 분류
  - 계절 매력형
  - 전망/사진형
  - 가족/인프라형
  - 힐링형
  - 숨은 명소형
